


# ShopDirector 产品设计文档体系治理规范

## 1. 文档体系目标

ShopDirector 产品设计文档体系用于建立统一的产品战略认知、设计边界和演进路径。

12份核心文档不是独立文档集合，而是一套具有明确职责、依赖关系和信息流转关系的产品知识体系。

核心原则：

- 每个问题只有一个主要归属文档；
- 下游文档继承上游战略，不重新定义上游内容；
- 避免多个文档重复描述同一概念；
- 所有产品设计必须能够追溯到产品战略。

# 4. 章节级职责治理原则

12份文档内部的每个章节，也必须定义明确职责，避免章节之间重复解释同一概念。

每个章节需要明确：

- Chapter Purpose：本章节解决什么问题；
- Input：本章节依赖哪些上游信息；
- Output：本章节输出给哪些下游文档；
- Boundary：本章节明确不负责什么。

核心原则：

一个概念只能拥有一个主要定义来源，其他文档只能引用或展开，不重新定义。

# 8. 统一章节模板

所有产品设计文档章节必须遵循统一结构。

## Chapter Purpose

说明：

本章节解决什么问题。

## Scope

说明：

本章节包含什么内容。

## Out of Scope

说明：

本章节明确不包含什么内容。

## Input

说明：

本章节依赖哪些上游信息。

## Output

说明：

本章节产生哪些结果，并提供给哪些下游模块。

## Core Principles

说明：

本章节遵循哪些核心原则。

## Decision Rules

说明：

本章节如何进行判断和取舍。

## Related Documents

说明：

与哪些文档存在依赖关系。

# 9. 文档治理最终原则

ShopDirector 产品文档体系遵循：

战略先于设计。

设计先于实现。

定义先于扩展。

边界先于功能。

每个文档负责回答一个核心问题。

每个章节负责一个明确主题。

每个概念拥有唯一来源。

其他文档只能引用、继承和展开，不允许重复定义。
